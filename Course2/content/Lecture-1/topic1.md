# Глава 1: Рекурсия

В рамках второй части курса мы переходим к более сложным концепциям построения алгоритмов. Начнем с фундаментального понятия — рекурсии.

1.  **Определение**:
    Рекурсия — это способ решения задачи, при котором функция вызывает сама себя. Это позволяет разбить сложную задачу на более простые подзадачи того же типа.

2.  **Структура рекурсивной функции**:
    Любая корректная рекурсивная функция должна содержать два обязательных компонента:
    * **Базовый случай (Base Case):** Условие, при котором функция прекращает вызывать саму себя и возвращает конкретное значение. Без этого произойдет переполнение стека (Stack Overflow).
    * **Рекурсивный шаг:** Вызов функции с измененными аргументами, приближающими нас к базовому случаю.

-----

### Пример: Факториал

Классический пример императивного и декларативного подхода через рекурсию. Факториал числа $N$ ($N!$) — это произведение всех натуральных чисел от 1 до $N$.

$$
N! = 
\begin{cases} 
1, & \text{если } N = 0 \\
N \times (N-1)!, & \text{если } N > 0 
\end{cases}
$$

Код на C:

```c
#include <stdio.h>

unsigned long long factorial(int n) {
    // Базовый случай
    if (n == 0 || n == 1) {
        return 1;
    }
    // Рекурсивный шаг
    return n * factorial(n - 1);
}

int main() {
    int num = 5;
    printf("%llu", factorial(num)); // Выведет 120
    return 0;
}
